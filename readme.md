# mxtest-cli

Mendix + Playwright testing CLI. This tool helps build Mendix docker images, start containers, run Playwright tests and produce HTML reports.

## Features
- Verify environment (mx CLI, Docker, Playwright)
- Initialize a test folder with a sample Playwright test
- Build Mendix Docker images and start services
- Run Playwright tests and generate HTML reports
- Manage docker compose, logs, status
- Manage `mxtest.config.json` settings

## Installation

```powershell
cd C:\Projects\custom-cli\mxtest-cli
npm install
npm link
```

After `npm link` the `mxtest` command will be available globally.

## Quickstart

1. Check your environment:

```powershell
mxtest doctor
```

2. Initialize tests (detects `.mpr` and creates `tests/sample.spec.js`):

```powershell
mxtest init
```

3. Build and start the application using Docker (optional ports):

```powershell
mxtest build 8080 5432
```

4. Run tests (from project root):

```powershell
mxtest test
```

5. Show the last Playwright report:

```powershell
mxtest result
```

## Configuration

The CLI stores its settings in `mxtest.config.json` at the project root. To view:

```powershell
mxtest config show
```

To update a setting (allowed keys: `testDir`, `appUrl`, `clientPort`, `postgresPort`, `waitTimeout`):

```powershell
mxtest config set clientPort 9090
```

## Templates

Templates are under `src/templates/` and will be used to create `.env`, `docker-compose.yml`, and a sample test file during `mxtest init` or `mxtest build`.
